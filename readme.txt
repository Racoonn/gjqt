整合过程中的一些小心得，希望能对你们以后的团队合作项目有帮助:
  1.习惯建立自己的文件夹。比如说我写index页面，约定好图片都放在assets/img/下，我的index页面需要的图片就放在assets/img/index/下，而不是直接散放在assets/img/下；组件也是的，在components下为每个页面建立一个文件夹（即使只有一个组件），通用的页头页尾也最好放在header、footer文件夹下。
  2.组件的样式必须加上scoped，示例：<style scoped></style>,不然会影响其他页面的样式。
  3.铺满整个页面的样式不要用固定宽度，要用width:100%。
  4.之前约定说后台用nodejs，但是果果用的php。以后团队约定好的规定，你们应该不会说不遵守吧。。。
